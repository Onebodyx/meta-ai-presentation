# Chess Coach — краткое задание на MVP

## Цель

Сделать **локальный учебный сервис**: пользователь играет против шахматного движка; после ответа движка интерфейс показывает **несколько сильных продолжений** из расчёта и **краткие пояснения** к ним через **облачный LLM API**. Фокус — понимание сильных идей в позиции, а не соревнование с человеком за рейтинг.

## Стек первой версии

- **Python** — язык приложения.
- **Streamlit** — веб-интерфейс для ввода ходов и отображения ответа движка, вариантов и текста объяснений.
- **Stockfish** (протокол **UCI**) — легальность ходов, сила игры движка, оценка позиции, формирование списка кандидатов продолжений (например режим **multiPV** или эквивалент в выбранной интеграции).
- **Облачный LLM API** — только генерация понятного текста по уже посчитанным данным; конкретный провайдер и формат запросов определяются на этапе реализации.

## Роли компонентов

| Компонент | Ответственность |
|-----------|-----------------|
| **Stockfish** | Проверка легальности ходов, выбор и исполнение хода движка, числовые/шахматные оценки позиции, перечень сильных продолжений для отображения. Это **единственный источник шахматной истины** в MVP. |
| **LLM** | Краткое **человекочитаемое объяснение** переданных ему вариантов и оценок. Не предлагает ходы самостоятельно и не заменяет расчёт движка. |

## Вне объёма MVP (явные ограничения)

Не входит в первую версию:

- регистрация пользователей и учётные записи;
- база данных и сохранение партий между сеансами;
- деплой в продакшен, хостинг, CI/CD как обязательная часть задачи;
- импорт/экспорт **PGN**, разбор баз партий;
- мультиплеер и игра человек–человек;
- продвинутые режимы тренировок (дебютные деревья, тактические генераторы и т.п.) и любая функциональность сверх описанного сценария.

## Пользовательский сценарий (MVP)

Пользователь запускает приложение локально, видит позицию и делает ход в интерфейсе; **Stockfish** проверяет ход, играет ответ и обновляет оценку. Затем система показывает **несколько сильных продолжений** из расчёта движка и запрашивает у **LLM** короткие объяснения этих вариантов; текст выводится рядом с данными движка. Цикл повторяется до окончания партии или сброса позиции (если такой элемент будет в интерфейсе минимально).

## Критерии готовности этого шага

- В корне проекта есть **`task-brief.md`** и **`project-concept.md`**.
- По документам однозначно понятны **границы MVP**, **разделение ролей Stockfish и LLM** и **стек v1**.
- Реализация кода приложения, правила для Cursor (`.cursor/rules/`) и вспомогательные субагенты — **отдельные последующие шаги**, не часть данного артефакта.
